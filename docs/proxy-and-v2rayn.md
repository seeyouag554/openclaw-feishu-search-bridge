# v2rayN 与代理地址说明

这一节只讲一件事：

> Search Bridge 用的代理到底是什么，应该怎么配，为什么这样配。

---

## 1. 这个方案里使用的代理软件

Windows 端代理软件：

- **v2rayN**

这个项目当前验证通过的方案，就是用 Windows 上的 v2rayN 给 WSL 里的 Search Bridge 提供代理能力。

---

## 2. 为什么必须开启“允许来自局域网的连接”

这是最关键的设置之一。

如果不开这个选项，WSL 里的程序虽然跑在你本机上，但它访问 Windows 代理时，通常还是会被当成“外部连接”处理。

结果就是：

- 浏览器也许能翻
- v2rayN 也许看起来在工作
- 但 WSL 里的 Search Bridge 连不上代理

所以这里的要求很明确：

> **v2rayN 必须开启「允许来自局域网的连接」**

不然这套方案大概率跑不通。

---

## 3. 为什么代理地址是 `http://172.27.208.1:10808`

这个地址不是 OpenClaw 自己生成的，也不是固定写死的魔法地址。

它是由两部分组成的：

### `172.27.208.1`
这是当前环境里：

- **WSL 访问 Windows 宿主机时看到的地址**

也就是说，Search Bridge 运行在 WSL 里，它要访问的是 Windows 上运行的 v2rayN，所以不能直接写 Windows 软件里的“本机回环地址”思路去理解。

### `10808`
这是当前环境里：

- **v2rayN 开启的 HTTP 代理端口**

所以合起来：

```text
http://172.27.208.1:10808
```

表示的是：

> **WSL 里的 Search Bridge，通过这个地址，访问 Windows 宿主机上的 v2rayN HTTP 代理。**

---

## 4. 为什么不能直接写 `127.0.0.1:10808`

因为这个 `127.0.0.1` 是 WSL 自己的回环地址，不是 Windows 的。

在 WSL 里写：

```text
http://127.0.0.1:10808
```

通常表示访问的是：

- WSL 自己内部的 10808 端口

而不是 Windows 侧 v2rayN 的代理端口。

所以这个方案里，Search Bridge 访问代理时，应该写的是：

- **Windows 宿主机在 WSL 视角下的地址**
- 而不是 WSL 自己的 `127.0.0.1`

---

## 5. 这个地址是不是所有电脑都一样

不是。

这个仓库里写的：

```text
http://172.27.208.1:10808
```

是当前这套环境里实测成功的地址。

不同机器上可能会变动的有两项：

### 1）WSL 看到的 Windows 宿主机地址
不一定都是 `172.27.208.1`

### 2）v2rayN 的 HTTP 代理端口
不一定都是 `10808`

所以你不能无脑照抄，最好先确认自己的环境。

---

## 6. 这个代理为什么只给 Search Bridge 用

这是整个项目最重要的设计之一。

### OpenClaw 主进程
职责：
- 飞书收消息
- 飞书回消息
- 模型推理
- 调工具

这个进程如果直接挂全局代理，经常会出现：

- 飞书能收不能回
- 飞书开放平台接口超时
- `bot/v3/info` 超时
- `tenant_access_token` 超时
- final reply failed

### Search Bridge
职责：
- 调 Brave Search API
- 抓取网页正文

这个进程天然就是“对外联网”的，所以它单独挂代理最合理。

所以最终结论就是：

> **OpenClaw 主进程不挂全局代理，Search Bridge 单独挂代理。**

---

## 7. 当前验证通过的代理配置

Search Bridge service 里当前验证通过的是：

```ini
Environment=HTTP_PROXY=http://172.27.208.1:10808
Environment=HTTPS_PROXY=http://172.27.208.1:10808
Environment=NO_PROXY=127.0.0.1,localhost,::1
```

这表示：

- 对外搜索请求走 v2rayN
- 本地回环地址不走代理

---

## 8. 当前方案的最简理解

你可以直接这么理解：

### 飞书链路
- 走直连
- 不挂全局代理

### 搜索链路
- 走 Search Bridge
- Search Bridge 单独挂 v2rayN 代理

一句话总结就是：

# 飞书直连，搜索单独代理
